# 考勤数据脱敏工具 - 示例数据模板

## 使用说明

这是一个示例模板,展示了考勤数据可能包含的字段。你可以根据实际数据格式调整。

## 推荐脱敏配置

| 字段名称 | 推荐脱敏方式 | 说明 |
|---------|------------|------|
| 姓名 | 部分脱敏 | 保留姓氏,如"张**" |
| 工号 | 部分脱敏 | 保留前后几位,如"00***123" |
| 手机号 | 完全脱敏 | 敏感联系方式 |
| 身份证号 | 完全脱敏 | 个人敏感信息 |
| 部门 | 不脱敏 | 用于统计分析 |
| 职位 | 不脱敏 | 非敏感信息 |
| 日期 | 不脱敏 | 考勤时间 |
| 上班打卡时间 | 不脱敏 | 工作时间记录 |
| 下班打卡时间 | 不脱敏 | 工作时间记录 |
| 工作时长 | 不脱敏 | 统计分析使用 |
| 迟到次数 | 不脱敏 | 统计分析使用 |
| 早退次数 | 不脱敏 | 统计分析使用 |
| 请假天数 | 不脱敏 | 统计分析使用 |

## 脱敏效果示例

### 部分脱敏
- 姓名: 张三 → 张*
- 工号: EMP001 → E**P1
- 部门: 技术部 → 技术*

### 完全脱敏
- 手机号: 13800138000 → ***********
- 身份证号: 110101199001011234 → *******************

### 哈希脱敏
- 工号: EMP001 → H7K2L9M4
- 姓名: 张三 → H8X3P5Y2

## 注意事项

1. **备份原始数据**: 在脱敏前务必备份原始文件
2. **测试验证**: 首次使用建议先用小样本测试
3. **检查字段**: 确保所有敏感字段都被正确配置脱敏规则
4. **导出前预览**: 导出前务必预览脱敏效果
5. **保存配置**: 如果有固定的脱敏需求,可以截图保存配置

## 数据安全建议

1. 敏感字段建议使用完全脱敏或哈希脱敏
2. 部分脱敏适用于需要保留一定可识别性的场景
3. 不脱敏字段要确保不包含敏感信息
4. 导出的脱敏文件仍需妥善保管,不要随意传播
