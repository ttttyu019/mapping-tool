# 自动应用映射表处理数据 - 使用指南

## 🎯 功能概述

这个功能允许你上传数据文件后,系统会**自动根据已有的映射表进行编码替换**,并导出处理后的文件。

适用于:已经有映射表,需要批量处理新数据的场景。

## 📋 使用场景

### 场景1: 每月考勤数据处理
1. **第一次建立映射表** (1月份)
   - 使用 mapping.html 为员工建立编码映射
   - 导出映射表保存

2. **后续月份自动处理** (2月、3月...)
   - 上传新的考勤数据
   - 系统自动根据已有映射表进行编码替换
   - 无需重复配置,一键处理

### 场景2: 多个文件统一编码
- 为第一批数据建立映射表
- 后续所有文件使用同一映射表处理
- 确保不同文件中相同人员使用相同编码

## 🚀 快速开始

### 步骤1: 确保已有映射表

首先需要建立映射表:
```
打开 mapping.html
→ 字段编码页面
→ 输入员工姓名、工号等数据
→ 生成编码
```

或者:
```
→ 导出导入页面
→ 批量数据编码
→ 批量生成映射表
```

### 步骤2: 自动处理新数据

1. 打开 mapping.html,切换到"导出导入"标签页
2. 在"自动应用映射表处理数据"部分:
   - 上传需要处理的数据文件 (Excel/CSV)
   - (可选) 配置字段映射
   - 选择是否自动生成新编码
3. 点击"自动处理并导出"
4. 下载处理后的文件

## ⚙️ 功能详解

### 1. 上传数据文件
- 支持 .xlsx, .xls, .csv 格式
- 文件大小建议不超过 10MB
- 确保文件格式正确,首行为列名

### 2. 字段映射配置 (可选)

**配置格式:**
```
列名 -> 字段类型

例如:
姓名 -> name
手机号 -> phone
工号 -> emp_id
```

**字段类型列表:**
- `name`: 姓名
- `phone`: 手机号
- `idcard`: 身份证号
- `emp_id`: 工号
- `department`: 部门
- `position`: 职位
- `other`: 其他

**使用建议:**
- 如果系统自动推断的字段类型不正确,手动配置
- 如果列名不标准,手动映射
- 每行一条映射规则

### 3. 自动生成新编码

**勾选此项时:**
- 遇到映射表中不存在的数据,自动生成新编码
- 新编码使用哈希算法(不可逆)
- 新编码会自动添加到映射表中

**不勾选时:**
- 只处理映射表中已有的数据
- 新数据保持原样
- 适用于只想对已知数据进行编码的场景

### 4. 预览映射关系

点击"预览映射关系"可以查看:
- 文件中的所有列名
- 系统推断的字段类型
- 每个字段是否有对应的映射表编码
- 列表示例值

**使用时机:**
- 上传文件后,先预览确认映射关系
- 发现字段类型推断错误时,使用字段映射配置修正

## 📊 处理流程

### 流程图

```
上传文件
    ↓
读取数据
    ↓
解析列名 → 字段类型推断
    ↓
逐行处理
    ├─ 查询映射表
    ├─ 找到编码 → 替换
    └─ 未找到 → 生成新编码(如果勾选)
    ↓
导出处理后的数据
    ↓
更新映射表
```

### 处理结果说明

系统会显示:
- 📊 总记录数: 处理了多少条数据
- 🔗 映射匹配: 使用已有编码的数量
- ➕ 新生成编码: 新创建的编码数量
- 💾 映射表更新: 映射表当前的记录总数

### 详细处理日志

处理完成后,可以查看详细日志:
- 每行的处理情况
- 哪些数据使用了已有编码
- 哪些数据生成了新编码
- 编码前后的对应关系

## 💡 实用技巧

### 技巧1: 先预览,后处理
上传文件后,先点击"预览映射关系",确认:
- 字段类型是否正确
- 哪些字段有映射表
- 示例值是否符合预期

### 技巧2: 字段映射配置优化

**标准列名 (自动推断正确):**
```
姓名、手机号、工号、部门...
```

**非标准列名 (需要手动配置):**
```
员工姓名 -> name
联系电话 -> phone
员工编号 -> emp_id
```

### 技巧3: 逐步建立映射表

**第一次处理:**
```
勾选"自动生成新编码"
→ 系统为所有数据生成编码
→ 建立完整的映射表
```

**第二次处理:**
```
不勾选"自动生成新编码"
→ 只处理映射表中已有的数据
→ 新数据保持原样
```

### 技巧4: 批量处理多个文件

使用同一映射表:
```
1. 处理第一个文件(生成映射表)
2. 依次处理其他文件(使用已有映射表)
3. 确保不同文件中相同人员编码一致
```

## 🔧 常见问题

**Q: 为什么有些数据没有被编码?**
A: 可能原因:
1. 映射表中没有该数据的编码
2. 没有勾选"自动生成新编码"
3. 字段类型推断错误

**Q: 如何检查字段类型是否正确?**
A: 使用"预览映射关系"功能,查看每列的字段类型和是否有映射表编码。

**Q: 新生成的编码会保存吗?**
A: 会,新编码会自动添加到映射表中,下次处理时会使用相同的编码。

**Q: 可以只对某些列进行编码吗?**
A: 可以,只有映射表中存在的字段类型才会被编码,其他列保持原样。

**Q: 处理后的文件格式会改变吗?**
A: 不会,只替换列中的数据,文件结构和列名保持不变。

**Q: 如何确保不同文件使用相同编码?**
A: 使用同一个映射表处理所有文件,相同的原始值会得到相同的编码。

## ⚠️ 注意事项

1. **备份数据**: 处理前务必备份原始文件
2. **验证结果**: 处理后检查几条数据确认编码正确
3. **映射表管理**: 定期清理不再使用的映射
4. **字段类型**: 确认字段类型正确,避免编码错误
5. **文件格式**: 确保上传的文件格式正确,首行为列名
