# 考勤数据本地脱敏系统

## 📋 系统组件

本系统包含两个核心工具:

### 1. **index.html** - 数据脱敏工具
   - 对原始数据进行脱敏处理
   - 支持完全脱敏、部分脱敏等方式
   - 输出脱敏后的数据文件

### 2. **mapping.html** - 转码解码映射表管理系统
   - 建立原始值和编码值的映射关系
   - 支持多种编码规则
   - 提供查询、导出、批量处理功能

## 🎯 完整工作流程

### 场景一: 只需脱敏,不需要对应关系
使用 `index.html` 进行数据脱敏即可

### 场景二: 需要保留数据对应关系

#### 步骤1: 建立映射表
1. 打开 `mapping.html`
2. 选择需要保留对应关系的字段(如工号、姓名)
3. 使用"批量数据编码"功能为这些字段生成编码
4. 导出映射表

#### 步骤2: 数据脱敏
1. 打开 `index.html`
2. 上传原始考勤数据
3. 对敏感字段配置脱敏规则
4. 导出脱敏后的数据

#### 步骤3: 数据分析
使用脱敏后的数据进行分析,需要时通过 `mapping.html` 查询原始值

## ⚡ 新增功能

### 自动应用映射表处理数据
上传数据文件后,系统会自动根据已有的映射表进行编码替换,并导出处理后的文件。

**特点:**
- 智能字段类型识别
- 支持自定义字段映射
- 可选择是否自动生成新编码
- 提供详细的处理日志
- 预览功能,提前确认映射关系

**适用场景:**
- 月度/季度数据批量处理
- 多文件统一编码
- 基于现有映射表处理新数据

详细使用方法请查看 **AUTO_PROCESS_GUIDE.md**

- ✅ **纯本地运行** - 所有数据处理都在浏览器中完成,数据不会上传到任何服务器
- ✅ **支持多种格式** - 支持 Excel (.xlsx, .xls) 和 CSV 文件
- ✅ **灵活脱敏** - 支持三种脱敏方式:
  - 部分脱敏: 保留首尾字符,中间用 * 替换
  - 完全脱敏: 全部替换为 *
  - 哈希脱敏: 不可逆的哈希转换
- ✅ **实时预览** - 可对比查看原始数据和脱敏后数据
- ✅ **一键导出** - 导出脱敏后的 Excel 文件

## 🚀 使用方法

### 1. 打开工具

双击 `index.html` 文件,或右键选择浏览器打开

### 2. 上传文件

- 直接拖拽文件到上传区域
- 或点击上传区域选择文件

### 3. 配置脱敏规则

- 系统会自动识别所有字段
- 为每个字段选择脱敏方式:
  - **不脱敏**: 保持原样
  - **部分脱敏**: 保留首尾,适合姓名、工号等
  - **完全脱敏**: 全部替换,适合手机号、身份证等
  - **哈希脱敏**: 不可逆转换,适合需要统计但不可还原的数据

### 4. 预览效果

- 点击"预览脱敏效果"按钮
- 可在"原始数据"和"脱敏后数据"标签页之间切换查看
- 脱敏后的字段会以黄色背景高亮显示

### 5. 导出数据

- 点击"导出脱敏数据"按钮
- 系统会自动下载脱敏后的 Excel 文件
- 文件名自动添加"_脱敏"后缀

## 🔒 安全说明

1. **完全本地化处理**: 所有数据都在浏览器内存中处理,不会上传到任何服务器
2. **不留痕迹**: 关闭浏览器后,所有处理的数据会自动清除
3. **不可逆设计**: 哈希脱敏方式确保数据无法被还原

## 💡 使用建议

1. **首次使用建议先用小文件测试**,确保脱敏效果符合预期
2. **导出前务必预览**,检查敏感字段是否都被正确脱敏
3. **保存原始文件**,脱敏后的数据无法还原
4. **定期备份**,防止意外数据丢失

## 📝 注意事项

- 支持的最大文件大小取决于浏览器内存限制
- 建议单次处理数据量不超过 10 万行
- 预览仅显示前 100 行数据,但导出会包含全部数据
- 确保使用现代浏览器 (Chrome, Edge, Firefox, Safari 最新版)

## 🆘 常见问题

**Q: 数据会泄露吗?**
A: 不会,所有处理都在本地浏览器完成,不会上传到任何服务器。

**Q: 可以批量处理多个文件吗?**
A: 当前版本支持单文件处理,多个文件需要逐个处理。

**Q: 脱敏后的数据能还原吗?**
A: 部分脱敏和完全脱敏理论上可以还原,但哈希脱敏是不可逆的。建议在脱敏前保存原始文件。

**Q: 支持哪些文件格式?**
A: 支持 .xlsx, .xls 和 .csv 格式的文件。

**Q: 可以自定义脱敏规则吗?**
A: 当前版本支持三种预设脱敏方式,如需自定义规则可以联系开发人员。
