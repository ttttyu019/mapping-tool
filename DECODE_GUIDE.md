# 转码解码系统使用指南

## 📋 系统概述

这是一个完整的转码解码管理系统,用于在脱敏数据处理时建立原始值和编码值的映射关系,确保在后续分析时能够对应回原始数据。

## 🎯 核心功能

### 1. 字段编码
将原始敏感数据转换为编码值,建立映射关系

### 2. 解码查询
通过编码值查询原始数据,或通过原始数据查询编码值

### 3. 映射表管理
查看、编辑、删除所有映射记录

### 4. 批量处理
支持批量编码数据文件,并自动生成映射表

## 📝 使用流程

### 场景一: 单个字段编码

**步骤:**
1. 打开 `mapping.html`
2. 选择"字段编码"标签页
3. 选择字段类型(姓名/手机号/身份证号/工号等)
4. 在文本框中输入原始数据(一行一个或用逗号分隔)
5. 选择编码规则
6. 点击"生成编码"按钮

**示例:**
```
字段类型: 姓名
原始数据:
张三
李四
王五

编码规则: 顺序编码
```

**结果:**
- 张三 → NAME0001
- 李四 → NAME0002
- 王五 → NAME0003

### 场景二: 批量编码考勤数据

**步骤:**
1. 打开 `mapping.html`
2. 切换到"导出导入"标签页
3. 在"批量数据编码"部分:
   - 上传考勤数据文件(Excel/CSV)
   - 指定需要编码的列(例如: 姓名,手机号,工号)
4. 点击"批量编码并下载"
5. 系统会自动:
   - 为指定列的所有数据生成编码
   - 自动建立映射关系
   - 下载编码后的数据文件

### 场景三: 查询解码

**步骤:**
1. 打开 `mapping.html`
2. 切换到"解码查询"标签页
3. 在搜索框输入:
   - 原始值: 张三
   - 或编码值: NAME0001
4. 点击"搜索"按钮
5. 查看查询结果

### 场景四: 导出映射表

**步骤:**
1. 打开 `mapping.html`
2. 切换到"导出导入"标签页
3. 选择导出格式:
   - Excel: 完整映射表
   - CSV: 便于其他系统使用
   - JSON: 便于程序处理
4. 点击"导出映射表"

## 🔐 四种编码规则

### 1. 顺序编码
- **格式**: 字段类型+数字
- **示例**: NAME0001, NAME0002...
- **适用**: 需要保持顺序关系的数据
- **特点**: 可读性强,便于人工识别

### 2. 随机编码
- **格式**: 8位随机字母数字组合
- **示例**: K3X8P2M4
- **适用**: 完全随机化的场景
- **特点**: 不可预测,安全性高

### 3. 哈希编码
- **格式**: H开头的哈希值
- **示例**: H7K2L9M4
- **适用**: 不可逆转换场景
- **特点**: 相同值编码相同,不可还原

### 4. 前缀编码
- **格式**: 自定义前缀+数字
- **示例**: EMP0001, USR0001...
- **适用**: 需要自定义编码规范
- **特点**: 灵活性高,可定制

## 📊 完整工作流程示例

### 阶段一: 数据脱敏处理

1. **使用 index.html 脱敏考勤数据**
   - 上传原始考勤数据
   - 配置脱敏规则(完全脱敏/部分脱敏)
   - 导出脱敏后的数据文件

2. **使用 mapping.html 建立映射表**
   - 批量编码需要保留对应关系的字段
   - 系统自动生成映射表
   - 导出编码映射表

### 阶段二: 数据分析

1. **使用脱敏后的数据进行分析**
   - 所有敏感信息已被替换
   - 可以安全地进行统计和分析
   - 可以分享给团队成员

2. **需要查看原始数据时**
   - 使用 mapping.html 查询映射表
   - 通过编码值查找原始值
   - 保留数据可追溯性

### 阶段三: 报告生成

1. **使用编码数据生成报告**
   - 统计结果使用编码值
   - 保留分析结果的可验证性

2. **如需展示原始数据**
   - 通过映射表进行对应
   - 确保数据来源可追溯

## 🔒 安全最佳实践

### 1. 映射表存储
- **本地存储**: 映射表保存在浏览器 LocalStorage
- **定期备份**: 定期导出映射表备份
- **安全保管**: 映射表文件要妥善保管

### 2. 访问控制
- **授权访问**: 只有授权人员才能访问映射表
- **使用后清理**: 分析完成后及时清除敏感映射
- **分级管理**: 不同敏感级别使用不同编码规则

### 3. 审计追踪
- **记录操作**: 记录所有编码和解码操作
- **时间戳**: 每条映射记录都有创建时间
- **可追溯**: 保留完整的映射历史

## 💡 实用技巧

### 技巧1: 批量处理

### 技巧3: 导出多种格式
- Excel: 便于人工查看和编辑
- CSV: 便于导入其他系统

### 技巧4: 搜索过滤
- 使用搜索功能快速定位特定数据
- 通过字段类型和编码规则筛选
- 支持模糊搜索

## 📁 文件说明

### index.html
- **功能**: 考勤数据脱敏工具
- **用途**: 对原始数据进行脱敏处理
- **输出**: 脱敏后的 Excel 文件

### mapping.html
- **功能**: 转码解码映射表管理系统
- **用途**: 建立编码和原始值的映射关系
- **输出**: 映射表(Excel/CSV/JSON)

### 完整流程
1. 使用 `index.html` 脱敏数据
2. 使用 `mapping.html` 建立需要保留对应关系的字段映射
3. 使用脱敏数据进行分析
4. 需要时通过 `mapping.html` 查询原始值

## ⚠️ 重要提醒

1. **备份数据**: 脱敏前务必备份原始数据
2. **测试验证**: 首次使用先用小数据测试
3. **安全保管**: 映射表文件要安全保存
4. **定期清理**: 不再需要的映射及时删除
5. **访问控制**: 限制映射表的访问权限

## 🆘 常见问题

**Q: 编码后会丢失原始数据吗?**
A: 不会,映射表会保存原始值和编码值的对应关系。

**Q: 可以多个项目共用一个映射表吗?**
A: 建议每个项目使用独立的映射表,避免混淆。

**Q: 映射表数据会过期吗?**
A: 不会过期,但建议定期清理不再使用的映射。

**Q: 编码可以自定义吗?**
A: 可以,使用前缀编码规则可以自定义编码前缀。

**Q: 如何确保数据安全?**
A: 映射表文件要加密保存,并限制访问权限。
